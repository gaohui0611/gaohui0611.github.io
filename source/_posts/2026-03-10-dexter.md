# Dexter：专为金融研究打造的 AI 自主代理

## 引言

在金融投资领域，研究分析是决策的基础。但面对海量的财报数据、市场信息和复杂的投资逻辑，即使是专业分析师也感到力不从心。今天，我们要介绍一个革命性的工具——**Dexter**，一个专门为金融研究设计的 AI 自主代理，它能够像专业分析师一样思考、规划、学习，并完成深度金融研究。

---

## 什么是 Dexter？

Dexter 是一个**自主金融研究代理**，它的定位非常明确：

> **"Think Claude Code, but built specifically for financial research."**
> （像 Claude Code 一样思考，但专为金融研究而构建。）

简单来说，Dexter 就是金融领域的 Claude Code。它能够：
- 自动分解复杂的金融问题
- 制定结构化的研究计划
- 实时获取市场数据
- 自我验证和迭代优化
- 最终输出数据支持的可靠答案

---

## 核心能力

### 1. 智能任务规划（Intelligent Task Planning）

Dexter 不是简单的一问一答，而是会像人类分析师一样思考：

**示例**：
```
用户提问："分析苹果公司过去 5 年的盈利能力趋势"

Dexter 的思考过程：
1. 获取苹果公司的年度利润表（过去 5 年）
2. 计算关键盈利指标（毛利率、净利率、ROE 等）
3. 分析收入和成本的变化趋势
4. 对比行业平均水平
5. 识别异常波动并分析原因
6. 总结盈利能力的整体趋势
```

### 2. 自主执行（Autonomous Execution）

Dexter 会自动选择并执行合适的工具来获取数据：

| 工具 | 功能 | 使用场景 |
|:-----|:-----|:---------|
| `get_income_statements` | 获取利润表 | 分析收入、成本、利润 |
| `get_balance_sheets` | 获取资产负债表 | 分析资产、负债、权益 |
| `get_cash_flow_statements` | 获取现金流量表 | 分析现金流入流出 |
| `search_web` | 网络搜索 | 获取新闻、行业分析 |

### 3. 自我验证（Self-Validation）

Dexter 会检查自己的工作，并不断迭代直到任务完成：

```
第一轮分析：
- 发现数据缺失 2022 年 Q3
- 标记需要补充

第二轮分析：
- 补充缺失数据
- 重新计算指标

第三轮验证：
- 检查指标计算是否正确
- 确认趋势分析合理

最终输出：
- 数据完整的分析报告
```

### 4. 实时金融数据（Real-Time Financial Data）

Dexter 能够访问：
- ✅ **财务报表**：利润表、资产负债表、现金流量表
- ✅ **历史数据**：多年的财务数据
- ✅ **多维度分析**：年度、季度数据
- ✅ **多公司对比**：同行业公司对比

### 5. 安全机制（Safety Features）

为了防止无限循环和执行失控，Dexter 内置了：
- **循环检测**：检测是否陷入重复执行
- **步骤限制**：限制最大执行步骤数
- **超时机制**：防止长时间运行

### 核心组件

#### 1. 任务规划器（Task Planner）

```typescript
// 将复杂问题分解为子任务
interface Task {
  id: string;
  description: string;
  dependencies: string[];
  tools: string[];
  expectedOutput: string;
}

// 示例：分析公司盈利能力
const tasks: Task[] = [
  {
    id: "1",
    description: "获取过去 5 年的利润表",
    dependencies: [],
    tools: ["get_income_statements"],
    expectedOutput: "5 年的收入、成本、利润数据"
  },
  {
    id: "2",
    description: "计算盈利指标",
    dependencies: ["1"],
    tools: ["calculate_metrics"],
    expectedOutput: "毛利率、净利率、ROE"
  },
  {
    id: "3",
    description: "分析趋势",
    dependencies: ["2"],
    tools: ["analyze_trend"],
    expectedOutput: "盈利能力趋势分析"
  }
];
```

#### 2. 执行引擎（Execution Engine）

```typescript
// 执行工具调用
async function executeTool(toolName: string, args: any) {
  // 1. 调用工具
  const result = await tools[toolName](args);

  // 2. LLM 总结结果
  const summary = await llm.summarize(result);

  // 3. 记录到 scratchpad
  await logToScratchpad({
    type: "tool_result",
    toolName,
    args,
    result,
    summary
  });

  return summary;
}
```

#### 3. 自我验证器（Self-Validator）

```typescript
// 检查任务完成质量
async function validateTask(task: Task, result: any) {
  const validation = await llm.evaluate({
    task: task.description,
    expectedOutput: task.expectedOutput,
    actualOutput: result,
    criteria: [
      "数据完整性",
      "计算准确性",
      "逻辑合理性"
    ]
  });

  if (validation.score < 0.8) {
    // 需要重新执行或补充数据
    return { status: "needs_refinement", feedback: validation.feedback };
  }

  return { status: "completed", score: validation.score };
}
```

### 示例 1：公司基本面分析

```bash
# 用户提问

6. 生成综合分析报告
```

**输出示例**：
```markdown
# 特斯拉（TSLA）财务健康分析报告

## 关键发现
- **资产负债率**：从 2019 年的 78% 降至 2024 年的 45%
- **现金储备**：持有现金 $29B，流动性充足

## 风险点
1. 资本支出持续高位（年均 $7-8B）
2. 自由现金流波动较大

## 结论
特斯拉财务状况整体健康，但需关注资本效率。
```

### 安装与配置

```bash
# 1. 克隆仓库
git clone https://github.com/virattt/dexter.git
cd dexter

# 2. 安装依赖
bun install

# 3. 配置环境变量
cp env.example .env

# 编辑 .env 文件，添加 API keys
# OPENAI_API_KEY=your-openai-api-key
# FINANCIAL_DATASETS_API_KEY=your-financial-datasets-api-key
# EXASEARCH_API_KEY=your-exa-api-key

# 4. 运行
bun start
```

## 评估体系

### 自动化评估

Dexter 包含完整的评估套件：

```bash
# 运行全部评估
bun run src/evals/run.ts

# 随机采样评估
bun run src/evals/run.ts --sample 10
```

**评估指标**：
- **准确性**：答案是否正确
- **完整性**：是否覆盖所有要点
- **数据支持**：是否有数据 backing
- **逻辑性**：推理是否合理

### LLM-as-Judge

使用另一个 LLM 来评判 Dexter 的输出质量：

```typescript
interface Evaluation {
  question: string;
  expectedAnswer: string;
  actualAnswer: string;
  scores: {
    accuracy: number;    // 准确性 0-1
    completeness: number; // 完整性 0-1
    dataQuality: number;  // 数据质量 0-1
  };
  feedback: string;
}
```
