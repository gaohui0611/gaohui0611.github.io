# nanobot：4000 行代码打造的超轻量级 AI 助手

在 AI 助手领域，Clawdbot 以其强大的功能闻名，但 43 万+ 行的代码量让许多开发者望而却步。香港大学数据科学实验室（HKUDS）推出的 **nanobot** 给出了一个全新的答案：**仅用约 4000 行代码，实现 Clawdbot 的核心功能**，代码量减少 99%，却保留了完整的智能代理能力。

---

## 为什么选择 nanobot？

### 极简主义的设计理念

| 特性 | nanobot | Clawdbot |
|------|---------|----------|
| 代码量 | ~4,000 行 | 430,000+ 行 |
| 启动速度 | 秒级 | 分钟级 |
| 资源占用 | 极低 | 较高 |
| 学习曲线 | 平缓 | 陡峭 |
| 定制难度 | 容易 | 困难 |

nanobot 的核心价值在于**"做减法"**——去掉复杂的抽象层，保留最本质的功能，让开发者能够：
- **快速理解**代码逻辑
- **轻松修改**和扩展
- **秒级启动**，即时响应
- **低资源占用**，随处运行

## 快速上手：2 分钟拥有自己的 AI 助手

### 1. 安装

```bash
# 方式一：从源码安装（推荐开发者）
git clone https://github.com/HKUDS/nanobot.git
cd nanobot
pip install -e .

# 方式二：使用 uv（快速稳定）
uv tool install nanobot-ai

# 方式三：PyPI 安装
pip install nanobot-ai
```

### 2. 初始化配置

```bash
nanobot onboard
```

这会创建配置文件 `~/.nanobot/config.json`。

### 3. 配置 API 密钥

编辑 `~/.nanobot/config.json`：

```json
{
  "providers": {
    "openrouter": {
      "apiKey": "sk-or-v1-xxx"
    }
  },
  "agents": {
    "defaults": {
      "model": "anthropic/claude-opus-4-5"
    }
  },
  "webSearch": {
    "apiKey": "BSA-xxx"
  }
}
```

**获取 API 密钥：**
- [OpenRouter](https://openrouter.ai)（推荐，支持所有模型）
- [Brave Search](https://brave.com/search/api/)（可选，用于网页搜索）

### 4. 开始对话

```bash
# 单条消息
nanobot agent -m "What is 2+2?"

# 交互模式
nanobot agent
```

## 架构设计：极简中的优雅

### 核心组件

```
nanobot/
├── agent/           # 智能代理核心
│   ├── reasoning    # 推理引擎
│   ├── memory       # 记忆管理

│   └── whatsapp     # WhatsApp 集成

nanobot 采用**配置优先**的设计理念，所有功能都可通过 `config.json` 灵活配置：

```json

      "enabled": false
    }

## 进阶用法

### 本地模型部署（vLLM）

不想用第三方 API？可以用本地模型：

```bash
# 1. 启动 vLLM 服务
vllm serve meta-llama/Llama-3.1-8B-Instruct --port 8000

# 2. 配置本地提供商
```json
{
  "providers": {
    "vllm": {
      "apiKey": "dummy",
      "apiBase": "http://localhost:8000/v1"
    }
  },
  "agents": {
    "defaults": {
      "model": "meta-llama/Llama-3.1-8B-Instruct"
    }
  }
}
```

### Telegram 机器人配置

```bash
# 1. 找 @BotFather 创建机器人，获取 Token
# 2. 找 @userinfobot 获取你的 User ID
# 3. 配置 config.json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN",
      "allowFrom": ["YOUR_USER_ID"]
    }
  }
}

# 4. 启动网关
nanobot gateway
```

### 语音消息支持

配置 Groq 提供商后，Telegram 语音消息会自动转录：

```json
{
  "providers": {
    "groq": {
      "apiKey": "gsk_xxx"
    }
  }
}
```

## 技术亮点解析

### 1. 极简的 Agent Loop

nanobot 的 Agent Loop 非常简洁：

```python
# 伪代码示意
while True:
    user_input = get_input()
    context = build_context(memory, user_input)
    response = llm.generate(context, tools)

    if response.has_tool_calls:
        results = execute_tools(response.tools)
        memory.add(results)
    else:
        send_reply(response.content)
        break
```

### 2. 模块化的工具系统

工具通过简单的装饰器注册：

```python
@tool("web_search")
def search_web(query: str) -> str:
    """搜索网页获取信息"""
    return brave_search(query)

@tool("market_analysis")
def analyze_market(symbol: str) -> dict:
    """分析市场数据"""
    return fetch_market_data(symbol)
```

### 3. 灵活的记忆管理

支持多种记忆后端：
- **内存**：快速但重启丢失
- **SQLite**：持久化本地存储
- **向量数据库**：语义检索（可选）

## 给开发者的启示

nanobot 的设计给我们几个重要启示：

### 1. 复杂性不等于价值

Clawdbot 的 43 万行代码固然强大，但 nanobot 证明：**4000 行代码也能实现核心功能**。在软件开发中，"足够好"往往比"完美"更有价值。

### 2. 代码即文档

nanobot 的代码如此简洁，以至于**阅读源码就是最好的学习方式**。这种透明性让开发者能够快速理解、修改和扩展。

### 3. 配置优于约定

通过 `config.json` 驱动所有功能，nanobot 实现了极高的灵活性，而无需修改代码。

### 4. 渐进式复杂度

从最简单的命令行对话开始，逐步添加 Telegram、WhatsApp、本地模型等高级功能，让用户体验平滑的学习曲线。
